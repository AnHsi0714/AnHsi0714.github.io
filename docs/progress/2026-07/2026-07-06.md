### 移植金屬碰撞（Metal Collision）

- `sketches/metalcollision.ts`：Matter.js 2D 物理——每 20 幀掉一個 3~7 邊多邊形（HSB 色相跟出生 x 位置走）進地板＋左右牆圍成的池子，每個多邊形帶一顆跟著旋轉的半圓眼睛，滑鼠可抓取拖曳（被抓住變灰、眼睛瞪成全圓）。
- **matter-js 改走 npm**：原稿在 OpenProcessing 用 `<script>` 注入 CDN 的 0.20.0，改成正式依賴（`matter-js` + `@types/matter-js`）。原稿 setup 最後的 `Matter.Engine.run()` 在 0.20 已移除（原稿那行其實會丟錯、只是位置在最後所以無害），移植只保留 draw() 裡的 `Engine.update()`。
- **hiDPI 抓取座標修正**：p5 canvas 內部像素是 CSS 尺寸乘 `pixelDensity`，Matter 的 `Mouse` 要靠 `data-pixel-ratio` 屬性換算，不設的話 retina 螢幕上抓取位置會偏一倍（原稿在 OpenProcessing 上也有這個問題，移植時修掉）。
- **出界清理**（移植後追加）：被甩出牆外的方塊會永遠留在引擎裡吃效能、堆太高冒出畫面頂端的也一樣，連續出界超過 120 幀（約 2 秒）就從物理世界＋繪製清單移除；靜態牆體與被滑鼠抓著的方塊除外。也順帶讓池子不會無限堆積（碰撞是 O(n²)）。
- **新增互動類型 `drag-physics`**（抓取拖曳物理物件）；原稿沒有存檔鍵，補上與其他作品一致的 S 儲存。
- `artworks.json` 的 `metal_collision` 圖檔副檔名修正（`.jpg` → `.png`，實際檔案是 png）。

### 畫廊列表頁：篩選／排序＋展場版面重整

- **篩選面板**（仿 `Articles.tsx` 的模式）：header 下方「篩選 / 排序」按鈕展開下拉面板，啟用條件數顯示在按鈕上、點外面收合。條件：標題關鍵字、起訖日期（作品 YYMMDD 轉 ISO 才能跟 `<input type="date">` 比較）、互動類型標籤 chips——畫廊版的「分類」直接取自 sketches 註冊表的 `interactions` 宣告，尚未移植的作品歸為「靜態展示」pseudo-tag，標籤只列實際有作品的。排序：最久／最新，預設最久（維持展牆從最早走到最新的動線）。
  - **配套修正**：滾輪轉向與「誰在中央就點亮誰」兩個 effect 原本只在掛載時抓一次 DOM，展位會隨篩選增減後改成依賴篩選後清單重掛 listener；篩到空集合顯示 `EmptyState`。
  - **切換排序跳回第一件**：如果留在原捲動位置，牆面在腳下重新洗牌、置中的常常還是同一件（最舊↔最新切換尤其明顯），看起來像按了沒反應。改成排序一換就 `scrollIntoView` 到新順序第一件（`behavior: "instant"`）；用 ref 記上一次排序值、只在真的切換時捲動，避免首次進場蓋掉 focusSlug「回到剛剛看的那件」。
- **展場版面重整**（畫作置中、捲軸貼底、消除頁面垂直捲軸）：
  - `.room` 高度鎖成正好一個視窗高（`height: 100vh` + `100dvh`）：main 的上下 padding 都被負 margin 抵銷，頁面總高＝房間高＝視窗高，右側不再出現頁面垂直捲軸。
  - `.header` 改成 `position: absolute` 浮在展牆上層、不佔版面高度（`pointer-events: none` + 子元素 `auto`，避免整條隱形帶擋住展牆滾輪），scroller 因此撐滿整個房間——垂直置中的基準變成整個視窗，橫向捲軸也剛好落在視窗底邊。
  - **畫框中心對齊視窗正中**：`align-items: center` 置中的是整組展位，但展位內畫框上方（燈 92 + 畫框 margin 108 = 200px）比下方（銘牌 ~84px）多 116px，scroller 下緣墊同樣的量，畫框中心就剛好落在視窗垂直中線，且與畫作高矮無關（上下是固定量、變的只有畫框自己的高度）。
  - scroller 明確設 `overflow-y: hidden`：`overflow-x: auto` 會讓 overflow-y 的 visible 被升級成 auto，視窗太矮時寧可上下裁切也不要冒出第二條垂直捲軸。
